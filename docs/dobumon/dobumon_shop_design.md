# 怒武者（ドブモン）ショップシステム詳細設計書

## 1. 概要
本システムは、怒武者の育成、交配、および戦闘能力に直接的・間接的な干渉を行うためのアイテム販売システムである。
インベントリ機能を持たず、購入した瞬間に効果が適用される「即時消費型」のデザインを採用している。

### 基本仕様
- **即時反映系**: 購入した個体のステータスや寿命、トレーニング回数を即座に変更する。
- **次動予約系**: `shop_flags` にデータを格納し、次回の「交配」「戦闘」等のアクション時に自動的に消費・反映される。
- **継続バフ系**: 特定の回数や期間、トレーニング効率を上昇させる。

---

## 2. アーキテクチャ構成

システムは以下の 4 つの主要コンポーネントで構成される。

### 2.1 データモジュール (`dob_items.py`)
- `ShopItem` クラス: 価格、カテゴリ、効果型、値を定義。
- `Shop` クラス: ショップ名、説明、VIP属性、陳列アイテムを定義。
- アイテムリスト: 全13種類のアイテム定義を保持。

### 2.2 UIレイヤー (`dob_shop_view.py`)
- `discord.ui.View` を拡張したマルチステップ・ウィザード。
- ショップ選択 -> アイテム選択 -> 対象怒武者選択 -> 購入確定 のフローを提供。

### 2.3 サービスレイヤー (`dob_shop_service.py`)
- トランザクション処理（経済システムとの連携）。
- アイテムごとの具体的なロジック（ハンドラー）を実行。
- 購入成功時のDB保存処理。

### 2.4 エフェクト集計エンジン (`dob_shop_effect_manager.py`)
- ゲームロジック側（交配、育成、戦闘）が、現在適用されているショップ効果を問い合わせるための静的メソッド群。
- `shop_flags` の内容をパースし、最終的な補正値を算出する。

## 4. ロジック統合フロー

### 4.1 トレーニング (`dob_train.py`)
1. `DobumonShopEffectManager.get_training_modifiers(dobu)` を呼び出す。
2. 返された `multiplier` を基本上昇量に掛ける。
3. `ignore_diminishing_returns` が True の場合、高齢期や高ランクによる減衰計算をスキップする。
4. 予約回数制（ブースター）の場合、実行後に `shop_flags` のカウントを減少させる。

### 4.2 交配 (`dob_breeders.py`)
1. `DobumonShopEffectManager.get_breeding_bonuses(p1, p2)` を呼び出し、両親分の補正を合算。
2. 継承IVの計算時に合算された `iv_bonus_base / high_iv_bonus` を加算。
3. 突然変異判定時に `mutation_chance_delta` を加算。
4. 禁忌深度の計算時に `taboo_depth_add` を加算。
5. 生成された子の `shop_flags` は初期化（空）状態で保存される。

### 4.3 戦闘 (`dob_calculator.py`)
1. `DobumonShopEffectManager.get_combat_modifiers(dobu)` を呼び出す。
2. 戦闘開始時の実ステータス（ATK/DEF/SPD/EVA）に補正倍率を適用。
3. 忌血の触媒等の「次動予約」フラグがある場合、戦闘終了後にフラグを消去する。

## 5. 設計上の注意点
- **エラーハンドリング**: ポイント不足や対象怒武者の不在、所持上限などのチェックは `DobumonShopService` ですべて事前検知する。
- **フラグの排他制御**: 同じキーの `shop_flags` を複数回購入した場合、基本的には「上書き」または「合算」されるが、アイテムごとのハンドラーでその挙動を制御する。
- **寿命の最小値**: アイテムによる寿命減少で個体が即死しないよう、最小値を 1 に制限している。
