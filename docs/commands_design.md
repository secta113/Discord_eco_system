# Discord Economy System - コマンド一覧と設計思想

## 1. アプリケーション設計思想 (Design Philosophy)

当システムのアーキテクチャとUIは、以下の思想に基づいて整理されています。

### ① UI/UXの最適化（操作のボタン統合と配置統一）
スラッシュコマンドはあくまで「募集の起点」にすぎないという考え方のもと、ゲームの「参加」「開始」「振る」「結果報告」など、進行上のアクションはすべてDiscordの直感的な**ボタンUIに統合**しています。
さらに、ユーザーの誤操作を防ぎ直感的にプレイできるよう、**全ゲーム共通で標準的なボタン配置（参加 → 進行 → キャンセル）と色分けを定義**しています。

*   **1番目（左）:** `✋ 参加する` `緑(Success)`
*   **2番目（中）:** `▶️ 開始する` `🏁 結果報告` `青(Primary)`
*   **3番目（右）:** `🛑 キャンセル` `赤(Danger)`

### ② フェイルセーフ（安全装置の独立）
UIボタン化を進める一方、Botプロセスの再起動に伴い過去のボタンが「インタラクションに失敗しました」状態に陥るリスクに対処しています。
万が一GUI（ボタン）からの操作が不可能になった場合でも、状態がロックされた「ゴースト募集」から安全に脱出し、ユーザーに全額返金を行うための最終防衛ラインとして、例外的に独立したコマンド `/dd-cancel` を維持しています。

### ③ 体系的なグループ分け（Sub-commandsの活用）
数多くのコマンドが煩雑に並ぶのを防ぐため、Discordのコマンドグループ機能を活用しています。
「ゲームの募集コマンド(`dd-game`)」、「ユーザー向けのお財布機能群(`dd-wallet`)」、「運営用の管理・修正機能群(`dd-mod`)」と機能の系統ごとに階層化し、サジェスト画面から目的の機能を探しやすく整理しました。

### ④ コマンドプレフィックスの選定 (`dd-` / Discord Duel)
システムのリニューアルに伴い、以前の `cc-` (Chinchiro Controller) から **`dd-`** への変更を行いました。

*   **プロジェクト名との親和性**: 新名称「Discord Duel」の略称。
*   **タイピングの効率性**: 同じキーを2回連続で叩くだけで入力が完了するため、モバイル・デスクトップの両環境において高速に呼び出しが可能です。

## 2. コマンド一覧

### 🎮 ゲーム募集コマンド (`dd-game` グループ)
*ゲームのセッションを作成するための「起点」となるコマンド。*

| コマンド | 説明 | 実行権限 |
| :--- | :--- | :--- |
| /dd-game chinchiro | `bet` (100) | チンチロリンの参加募集を開始します。 | 全員 |
| /dd-game blackjack | `bet` (100) | ブラックジャックの参加募集を開始します。 | 全員 |
| /dd-game match | `bet` (必須) | 外部ゲーム用のエスクロー（預かり金）マッチの募集を開始します。 | 全員 |
| /dd-game gacha | なし | 1日3回限定のデイリーガチャ。初回のみ未所持確定枠あり。 | 全員 |

### 🛡️ フェイルセーフ（安全装置）

| コマンド | 説明 | 実行権限 |
| :--- | :--- | :--- |
| `/dd-cancel` | **【安全装置】** 全コマンド共通の緊急脱出手段。現在募集中のゲーム状態・UIを強制終了し、参加者に全額を返金します。 | ホスト / 管理者 |

---

### 💳 経済・お財布コマンド (`dd-wallet` グループ)
*一般プレイヤーが自身の所持ポイント（リソース）を確認・運用するためのコマンド群。*

| コマンド | 引数（オプション） | 説明 |
| :--- | :--- | :--- |
| /dd-wallet balance | なし | 自分の現在の所持ポイントを確認します。 |
| /dd-wallet stats | なし | 自分のプロフィール（所持金、戦績、ガチャコンプ率）を表示します。 |
| /dd-wallet jackpot | なし | システム・ジャックポット（還元待機プール）の総額を確認します。 |
| /dd-wallet daily | なし | 1日1回のログインボーナス（デイリー）を受け取ります。 |
| /dd-wallet pay | `target`, `amount` | 別ユーザー（`target`）に対して、自分の残高からポイント（`amount`）を送金します。 |

---

### ⚙️ 運営・システム管理コマンド (`dd-mod` グループ)
*不正の取り締まり、イベント報酬の配布、デバッグなどを行うための管理者専用コマンド群。*

| コマンド | 引数（オプション） | 説明 |
| :--- | :--- | :--- |
| /dd-mod maintenance | `state` (on/off) | システムの緊急停止（新規ゲーム募集の制限）または再開を行います。 |
| /dd-mod logs | `target` | 指定ユーザーの直近のポイント加減振履歴を表示します。 |
| /dd-mod status | なし | ボットのバージョン、Uptime、全ユーザーの総流通ポイントを表示します。 |
| /dd-mod status_all | なし | 全ユーザーの残高と統計（勝敗数等）をランキング形式で表示します。 |
| /dd-mod winner | なし | 最多資産、最多勝利、最大一撃配当の「殿堂入り」プレイヤーを表彰（公開）します。 |

## 3. CLI管理ツール (`admintool.py`)
*Botを介さず、サーバーのターミナルから直接データベース操作を行うためのツール。*

| コマンド | 引数 | 説明 |
| :--- | :--- | :--- |
| `python admintool.py give` | `user_id`, `amount`, `--reason` | 特定ユーザーにポイントを付与します。 |
| `python admintool.py remove` | `user_id`, `amount`, `--reason` | 特定ユーザーからポイントを差し引きます。 |
| `python admintool.py transfer` | `from_id`, `to_id`, `amount`, `--reason` | ユーザー間でポイントを強制移動します。 |
| `python admintool.py balance` | `user_id` | 指定したユーザーの現在の残高を表示します。 |
