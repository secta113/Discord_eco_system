# ポーカー (Texas Hold'em) ゲーム仕様 (poker.md)

> **最終更新**: 2026-04-08  
> **新規作成**: v2.5 | **更新**: v2.10, **v2.11**, **v2.12**, **v2.15**

---

## 1. 概要

テキサス・ホールデム（Texas Hold'em）は、プレイヤーに配られる2枚の「ホールカード」と、全プレイヤー共通の5枚の「コミュニティカード」を組み合わせ、最強の5枚の手札を作るポーカーです。

当ボットでは、最大6人による対人戦（PvP）およびNPC補充によるソロプレイをサポートします。

---

## 2. ゲームフロー

1.  **募集 (`recruiting`)**: コマンド `/dd-game poker [bet] [buyin] [players]` で開始。
2.  **プリフロップ (`pre-flop`)**: 各プレイヤーに2枚のカードを伏せて配ります。
3.  **ベッティングラウンド 1**: 順番にアクション（チェック/コール/レイズ/フォールド）を行います。
4.  **フロップ (`flop`)**: コミュニティカードを3枚公開します。
5.  **ベッティングラウンド 2**
6.  **ターン (`turn`)**: コミュニティカードを1枚追加で公開します（計4枚）。
7.  **ベッティングラウンド 3**
8.  **リバー (`river`)**: コミュニティカードを1枚追加で公開します（計5枚）。
9.  **ベッティングラウンド 4**
10. **ショウダウン (`showdown`)**: 残ったプレイヤーで手札を公開し、最強の手を持つプレイヤーがポット（賭け金の総計）を獲得します。

---

## 3. 役判定と配当

### 3.1 役の強さ (降順)
1.  **ロイヤルフラッシュ**: 同じスートの A-K-Q-J-10
2.  **ストレートフラッシュ**: 同じスートの連続した5枚
3.  **フォー・オブ・ア・カインド (クワッズ)**: 同じ数字が4枚
4.  **フルハウス**: スリー・オブ・ア・カインド + ワンペア
5.  **フラッシュ**: 同じスートが5枚
6.  **ストレート**: 数字が連続した5枚 (A-2-3-4-5 は最弱のストレート)
7.  **スリー・オブ・ア・カインド (セット/トリップス)**: 同じ数字が3枚
8.  **ツーペア**: 同じ数字のペアが2組
9.  **ワンペア**: 同じ数字のペアが1組
10. **ハイカード**: 役なし。個別のカードの強さで比較

### 3.2 ジャックポット (Jackpot)
勝利したプレイヤーの役が以下の希少役である場合、ジャックポットプールから追加のボーナスが支払われます。  
配当は現在のプール残高 × 放出率と、セーフティ・キャップの小さい方が適用されます。

| 特殊役 | レアリティ | プール放出率 | セーフティ・キャップ |
| :--- | :--- | :--- | :--- |
| **ロイヤルフラッシュ** | LEGENDARY | **100%** | 100,000 pts |
| **ストレートフラッシュ** | EPIC | **30%** | 100,000 pts |
| **フォー・オブ・ア・カインド** | RARE | **10%** | 100,000 pts |

- ※キャップはボーナスの上限であり、プールが十分にあっても最大 100,000 pts までしか受け取れません。
- ※放出履歴は `jackpot_logs` テーブルに監査ログとして記録されます。

## 4. 特殊ルール

1.  **エフェメラル・ハンド**: 自分の手札（ホールカード）は、Embed内の「手札を確認」ボタンを押すことで、自分にだけ見えるメッセージとして確認できます。
2.  **ポットの分配**: 勝利者が複数いる場合（役が完全に一致）、ポットは均等に分配されます。
3.  **接待の除外**: 対人戦（PvP）の公平性を維持するため、テキサス・ホールデムでは接待アルゴリズムは発動しません。
4.  **NPC補充とAIランク制 (v2.13, v2.15 改訂)**: 
    ソロプレイや人数不足時の穴埋めとして出現するNPCは、個別抽選ではなく**「卓全体の設計図（ブループリント）」**に基づいて構成されます。
    - **出現確率の制御 (v2.15)**: 
        1ゲーム開始時に「卓のテーマ（Rank）」を 13%/30%/27%/20%/10% の比率で決定します。
        - **MONSTER (物怪)**: 出現率 13% 定。出現時は1卓につき1人固定。
        - **LEGENDARY (伝説)**: 出現率 30%。1卓につき最大3人までに制限。
        - **RARE/COMMON/TRASH**: 残りの確率を分配。
    - **テーブル・アーキタイプ (多様性の確保)**: 
        `data/poker_blueprints.json` に定義された組み合わせテンプレートから選択されます。
        「ボスの乱入」「プロ同士の決闘」「初心者の集い」など、卓ごとに多様なシチュエーションが生成され、ランクだけでなく性格やリスク許容度の傾向も卓のテーマに合わせて調整されます。
    - **経済的重み付け (v2.14)**:
        AIはアクションの際、平均スタックに対するベット額の割合を計算し、性格に応じて振る舞いを変えます。
5.  **NPCの行動ロジックと性格・リスク設定 (v2.16 補足)**:
    NPCは単なるランクだけでなく、14通りの「性格 × リスク許容度」の組み合わせによって、人間らしい多様な振る舞いを見せます。
    - **AIランク (難易度/格付け)**: 
        卓のテーマによって決定されるAIの基本性能です。
        - **MONSTER (物怪)**: 最上位。リバーでの「手札交換(Cheat)」能力を持ち、常に全プレイヤーの手札をピーピングする。極めて狡猾。
        - **LEGENDARY (伝説)**: プロレベル。期待値計算に基づく正確なプレイを行う。
        - **RARE/COMMON**: 中級〜一般的なプレイヤー。
        - **TRASH (トラッシュ)**: 初心者・カモ。無謀なコールや臆病なフォールドが目立つ。
    - **リスクレベル (0.0 〜 1.0)**: 
         NPCごとにランダムに割り振られる「胆力」の数値です。
        - **高い数値**: 強いブラフや、多少の不利を無視した攻撃的なベットを行いやすくなります。
        - **低い数値**: 慎重になり、リスクを避けて早めにフォールドする傾向が強まります。
    - **性格タイプ (Personality)**:
        | 性格 | 行動傾向 |
        | :--- | :--- |
        | **Aggressive** | レイズ頻度が高い。特にMonsterの場合は法外な額(5-10倍)を突っ込む。 |
        | **Timid** | 弱気。スタックに対する要求額が高いと、良い手でも降りやすい。 |
        | **Calculated** | 戦略的。Monsterの場合、序盤は泳がせ(Call)、終盤に牙を剥く(Raise)「潜伏」を行う。 |
        | **Shark** | 絞り込み。参加率は低いが、レイズした場合は非常に強力な手を持っている。 |
        | **Station** | コールステーション。フォールドを極端に嫌い、最後まで付いてきやすい。 |
        | **Bluffer** | ハッタリ屋。手札の強さに関わらず、唐突にレイズを仕掛ける。 |
        | **Normal** | 汎用的。ランクに応じた標準的な判断を行う。 |

6.  **インフレ防止（経済調整弁）**:
    - NPCのチップ原資はシステム供給。
    - **NPCが勝利して得た純利益はジャックポットプールへ蓄積。** 
    - **NPCの未使用スタックはシステムへ戻り消滅。** これによりポイントの過剰流通を防ぎます。

## 5. ベッティング・スタック制仕様 (v2.10)

参加時にウォレット残高からバイイン額をエスクロー（預かり）し、テーブル上の「スタック（持ち点）」として管理します。
ゲーム内のベットやレイズはすべてこの「スタック」内で完結し、終了時点で一括清算（払い戻し）が行われます。

- **SB/BB**: スモールブラインドとビッグブラインドを導入し、ゲーム開始時にスタックから強制的に徴収します（SBはBBの半額）。
- **バイイン額**: デフォルトは BB × 20。残高が足りない場合はショートバイイン（手持ち全額）で参加可能です。
- **最低参加条件**: BB相当のポイントを保有していること（BBを払えない場合は参加不可）。
- **固定レイズ**: 1回のレイズ額はBBと同額などの簡易固定額として処理します。
- **オールイン**: 賭け額に対してスタックが不足している場合、残りスタック全額をベットします（ショートオールイン対応）。
- **ゲーム終了時**: 残スタックは全員にウォレットへ返還。勝者には加えてポット分が加算されます。

## 6. ターン管理仕様 (v2.10)

### 6.1 ヘッズアップ（2人戦）の先行ルール
2人対戦では特殊な先行ルールが適用されます。
| フェーズ | 先行プレイヤー |
| :--- | :--- |
| プリフロップ | **SB（ボタン）** - UTG相当で先行 |
| フロップ〜リバー | **BB（非ボタン）** |

### 6.2 ターン進行・実装構造 (v2.12 リファクタリング反映)
- **ターン管理**: `loop_rotate_turn()` と `_ensure_valid_turn()` による循環管理とスキップ処理を継続。
- **アクション・カプセル化**: 各アクション（Fold, Call, Raise, All-in）のロジックを個別のプライベートメソッドに抽出し、`handle_action` をスリム化。
- **ベット更新の一元化**: `_set_new_max_bet()` により、最大ベット額の更新と他プレイヤーのアクションフラグの管理を統一的に実施。
- **モデルの静的化**: `PokerPlayer` クラスに `final_hand_name` などの属性を明示的に定義し、実行時の動的拡張を排除して堅牢性を向上。

### 6.3 UI側のボタン制御
旧バージョンでは「ボタンを押した本人のID」に基づいてUIボタンの有効・無効を制御し同期ズレを起こしていましたが、v2.10より「`session.get_current_player()` で取得した**現在の正規のターン保持者**」を基準にボタンの有効化を一律設定するアーキテクチャに修正しました。これにより他人のターンにアクションしてしまう同期バグを根絶しました。
