## 1. 背景とコンセプト

### 1.1 開発の背景

本プロジェクトは、Discordサーバー内のコミュニティ活性化を目的とした「経済＆ゲームBot」です。  
単なる乱数生成ゲーム（チンチロリン等）から出発し、ユーザー間の競争と交流を継続的に促進するために**永続的な通貨（ポイント）と経済圏**を構築する方向に発展しました。

**現在のテーマ：「年末の会食費用を決定するための1年にわたる長期決闘」**  
経済の停滞（硬直化）を防ぎ、最後まで逆転の可能性を維持する「飯代決闘シミュレーション」として設計されています。

### 1.2 アーキテクチャ目標

**Raspberry Pi 5 での常駐稼働**（discord.py Gateway API ベース）を最終的な運用形態とします。

1. **State Management**: WalletとゲームセッションをSQLiteで永続化し、再起動時にもデータを保持。
2. **Stateless Design**: Botプロセスがメモリに状態を抱え込まない設計で、再起動耐性を確保。
3. **Escrow Service**: ゲームの勝敗に基づく自動ポイント徴収・分配で公平な賭けを実現。

**【現在の実装ステータス】**  
コアとなる経済ロジック・ゲームエンジン・SQLiteによる状態永続化が実装済みで、Raspberry Pi 上で稼働中。

## 2. 機能スコープ

| 機能 | 概要 |
| :--- | :--- |
| **Economy System** | ポイント口座（Wallet）管理、送金、デイリーボーナス（セーフティネット型） |
| **Chinchiro** | 6面ダイス×3のロールシミュレーション・役判定・Winner Takes All |
| **Blackjack** | ディーラー(Bot) vs プレイヤー、8デッキ、希少役ジャックポット、**スプリット対応** |
| **Match (エスクロー)** | 外部ゲーム（麻雀・FPS等）の結果に基づく自動決済 |
| **Gacha** | 1日3回限定・コスト増分制・全250種の日常イベントコレクション |
| **Jackpot** | 希少役発生時にシステムプールから特別配当を支払う仕組み |

## 3. 技術スタック

| パラメータ名 | 値 | 説明 |
| :--- | :--- | :--- |
| デッキ数 | **8デッキ（416枚）** | 長期ゲームでもカウンティングを実質不可能にする |
| **スプリット (Split)** | 可能(1回) | 同じランクのペア時。A+10はBJではなく通常の21(1.0倍)扱い |
| **Framework** | `discord.py` (Gateway API) |
| **Database** | SQLite（WAL モード）|
| **Infrastructure** | Raspberry Pi 5 / 同等の常時稼働 Linux 環境 |
| **主要ライブラリ** | `discord.py`, `python-dotenv`, `pydantic`, `statistics` |
