# 怒武者野生戦システムおよびサービス構成のリファクタリング

## 概要
怒武者システムの保守性と拡張性を向上させるため、巨大な `DobumonService` (God Object) の解体と、野生戦システムのランク制への移行を実施しました。

## 実施した主な変更

### 1. サービスの専門分化 (Service Decomposition)
モノリシックな `DobumonService` を廃止し、以下の専門サービスに分割しました。
- **DobumonTrainingService**: 育成・特訓ロジックの担当
- **DobumonBreedingService**: 交配・遺伝ロジックの担当
- **DobumonMarketService**: 売却・命名・ショップロジックの担当
- **DobumonBattleService**: 野生戦・決闘（PvP）ロジックの担当

これらは `DobumonCog` の初期化時に個別に注入され、各 UI (View) に必要な分だけ渡される疎結合な設計となっています。

### 2. 野生戦システムの刷新 (Wild Battle Refactoring)
従来の動的なレベルスケーリングを廃止し、ランクベースの難易度システムを導入しました。
- **ランク制の導入**: Rank D〜S+ までの固定ステータスレンジを持つ難易度設定。
- **3ステップ・ウィザード**: 怒武者選択 -> 難易度選択 -> マップ選択 のフローを導入し、ユーザー体験を向上。
- **Forbidden Depth (禁忌の深度)**: 属性や特性に応じた戦略的なバフ・デバフ計算、および特殊アイテムによるボーナスの導入。

### 3. ディレクトリ構造の整理
`logic/dobumon/` 配下を機能ごとにサブパッケージ化し、ファイル単体の肥大化を防ぎました。
- `core/`: 基本モデル、例外、マネージャー、基盤サービス
- `dob_battle/`: 戦闘エンジン、野生戦、PvP、報酬精算
- `dob_views/`: 各種 UI (discord.ui.View)
- `training/`: 育成エンジン、コスト計算
- `genetics/`: 遺伝・交配ロジック、血統図生成

### 4. テストスイートの近代化
テストコードを役割ごとに整理し、実行効率と保守性を向上させました。
- `tests/unit/`: 単体テスト
- `tests/integration/`: Cog レベルの結合テスト
- `tests/simulations/`: 遺伝や戦闘バランスの統計的なシミュレーションテスト

## 技術的メリット
- **循環参照の解消**: サービスを分割し、インポートパスを整理したことで、依存関係のループを完全に排除しました。
- **疎結合化**: UI (View) が特定のサービスのみに依存する設計となり、ユニットテストにおけるモック化が容易になりました。
- **可読性の向上**: 各クラスが単一責任原則 (SRP) に従うようになり、メンテナンスコストが大幅に削減されました。
