# ブラックジャック (Blackjack) 詳細仕様

> **最終更新**: 2026-03-30  
> *v2.11 ジャックポット共通システムへの移行に基づき更新*

---

## 1. 基本ルール

- **目的**: カードの合計値を「21」に近づけ、ディーラーよりも高い点数を目指します。21を超えると「バースト（Bust）」となり、即座に敗北します。
- **使用デッキ**: **8デッキ（416枚）**。
  - カウンティング対策として、デッキ残数が少なくなると自動的にリシャッフルされます。
- **ディーラーの行動**: 合計値が **17点以上** になるまでカードを引き続けます。17点に達した時点で自動的に「スタンド（Stand）」します。
- **A（エース）の扱い**: 1点または11点として、より有利な方で計算されます。

---

## 2. プレイヤーのアクション

| アクション | 説明 |
| :--- | :--- |
| **Hit (ヒット)** | 山札からカードを1枚引きます。 |
| **Stand (スタンド)** | 現在の手札で勝負を確定させます。 |
| **Double Down (ダブル)** | 賭け金を **2倍** に増やし、カードを **1枚だけ** 引いて勝負します（最初の2枚の時のみ可能）。 |
| **Split (スプリット)** | 同じランクのペアを2つの独立した手札に分けます（詳細は後述）。 |

---

## 3. スプリット (Split) 仕様

同じポイントバリューのカードが2枚配られた場合、追加の賭け金を支払って手札を分割できます。

- **回数制限**: 1ゲームにつき **1回** のみ可能です。
- **実行条件**:
  - 手札がちょうど **2枚** であること。
  - 2枚の **ポイントバリュー（Value）** が一致していること（例：「10」と「J」の組み合わせでも可能）。
- **コスト**: 元の賭け金と同額を追加で支払います（賭け金が手札ごとに発生します）。
- **特殊ルール**:
  - **No BJ on split**: スプリット直後に21点（A + 10相当）になっても、「Blackjack（1.5倍利益）」ではなく「通常の21点（1.0倍利益）」扱いとなり、自動的にスタンドします。
  - **分割後のプレイ**: 1つ目の手札のアクションをすべて完了してから、2つ目の手札に移ります。
  - **追加入手後のアクション**: スプリット後の各手札に対し、通常の Hit / Stand / Double Down が可能です。

---

## 4. 配当と清算

| 結果 | 配当倍率 | 説明 |
| :--- | :--- | :--- |
| **Win (勝利)** | **2.0倍** | プレイヤー > ディーラー、またはディーラーがバースト。 |
| **Blackjack (BJ)** | **2.5倍** | 最初の2枚で21点（利益1.5倍）。スプリット後は対象外。 |
| **Push (引き分け)** | **1.0倍** | プレイヤー == ディーラー。賭け金がそのまま返還されます。 |
| **Lose / Bust** | **0倍** | ディーラー > プレイヤー、またはプレイヤーがバースト。没収された賭け金はジャックポットへ。 |

---

## 5. 希少役とジャックポット (Jackpot)

特定の役で勝利した場合、共有プール（システムアカウント `ID:0`）からボーナスが支払われます。  
配当はレアリティと現在のプール残高に基づき計算され、最大支払い額（セーフティ・キャップ）が設定されています。

| 役名 | レアリティ | プール放出率 | 賭け金倍率 | キャップ |
| :--- | :--- | :--- | :--- | :--- |
| **7-7-7 (Triple Seven)** | EPIC | **30%** | **21倍** | 50,000 pts |
| **6-Card Charlie** | RARE | **10%** | **7倍** | 50,000 pts |
| **6-7-8 (Sequential 21)** | COMMON | **3%** | **5倍** | 50,000 pts |
| **Suited Blackjack** | - | 固定 **1,000 pts** | 2.5倍相当 | - |

- ※ジャックポットの配当はシステムプール残高 × 放出率と、キャップの **小さい方** が適用されます。
- ※`Suited Blackjack` の固定ボーナスは、プール残高が不足している場合は支払われません。
- ※接待アルゴリズムが発動した手札では、ジャックポット当選権利が **無効化** されます。
- ※全ての放出履歴は `jackpot_logs` テーブルに監査ログとして記録されます。

## 6. 接待アルゴリズム (Hospitality)

プレイヤーの不快感軽減およびバースト回避のための介入ロジックです。

- **アルゴリズム詳細**: [接待ロジック詳細仕様 (hospitality_logic.md)](file:///d:/python_project/Discord_eco_system/docs/hospitality_logic.md) を参照。
- **発動タイミング**: 
  - プレイヤーのHit時のバースト回避（12〜16時）
  - ディーラーのバースト誘導（12〜16時）
- **ペナルティ**: 接待が発動した手札では、希少役（ジャックポット）の当選権利が **無効化** されます。

## 7. UI/UX

- **表示モード**:
  - **ソロモード**: 1名でプレイ中、カードとスコアを大きく表示。
  - **マルチモード**: 2名以上でプレイ中、全員の状態をリスト形式で表示。
- **デッキ残数表示**: 常に現在のデッキ残弾数がフッターに表示されます。
- **履歴記録**: すべての勝敗と配当は `stats` テーブルに記録され、ユーザーのプロフィールに反映されます。
